# ADR-003: 洋葱式中间件管道

## 状态

已采纳

## 背景

HTTP 请求需要经过多个处理阶段：认证、速率限制、日志记录、安全头注入等。需要选择一种中间件架构。

## 决策

采用洋葱式中间件管道，每个中间件包装内层函数：

```text
请求 → 安全头 → CORS → 内容类型验证 → 请求体限制 → 认证 → 速率限制 → 日志 → 应用处理
```

每个中间件是 `(app) => async (request) => payload` 形式的高阶函数。

## 后果

### 正面

- 中间件可独立测试
- 管道顺序清晰可控
- 新增中间件只需一行代码
- 中间件可在不同上下文中复用

### 负面

- 中间件顺序错误可能导致安全漏洞
- 深层嵌套可能影响调试体验
