# ADR-001: 零第三方运行时依赖

## 状态

已采纳

## 背景

Release Guardian 是一个企业级发布治理 API，需要在高安全性的生产环境中运行。传统 Node.js 项目通常依赖大量的第三方 npm 包，这带来了：

- 供应链攻击风险
- 许可证合规负担
- 依赖地狱和版本冲突
- 安全审计复杂度

## 决策

Release Guardian 的运行时代码不使用任何第三方 npm 包。所有功能均使用 Node.js 内置模块实现：

- `node:http` — HTTP 服务器
- `node:crypto` — UUID 生成、哈希、ETag
- `node:url` — URL 解析
- `node:fs` — 文件持久化
- `node:test` — 测试框架（仅开发）

## 后果

### 正面

- 供应链攻击面最小化
- 无许可证合规问题
- 安全审计简单明了
- 部署体积小
- 启动速度快

### 负面

- 需要手动实现某些功能（如 JSON 解析、文件原子写入）
- 开发速度可能较慢
- 某些高级功能需要更多代码

### 中性

- 团队需要对 Node.js 内置 API 有深入了解
- 测试覆盖率要求更高
