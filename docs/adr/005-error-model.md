# ADR-005: 统一错误响应模型

## 状态

已采纳

## 背景

API 需要一致的错误格式，便于客户端处理和调试。

## 决策

所有错误响应遵循统一格式：

```json
{
  "error": {
    "code": "validation_error",
    "message": "Human-readable description.",
    "details": {}
  }
}
```

- `code`：机器可读的错误标识符，用于程序化处理
- `message`：人类可读的错误描述
- `details`：可选的上下文信息（字段名、约束条件等）

## 后果

### 正面

- 客户端可通过 `code` 字段实现程序化错误处理
- `message` 字段便于开发调试
- 统一格式降低客户端集成成本

### 负面

- 需要维护错误码注册表
- 错误码变更可能影响客户端
