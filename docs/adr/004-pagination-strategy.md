# ADR-004: 偏移量分页策略

## 状态

已采纳

## 背景

列表 API 需要支持分页。常见方案有：

- 偏移量分页（offset/limit）
- 游标分页（cursor）
- 键集分页（keyset）

## 决策

采用偏移量分页（offset/limit），响应包含分页元数据：

```json
{
  "data": [...],
  "pagination": {
    "total": 100,
    "limit": 20,
    "offset": 0,
    "hasMore": true
  }
}
```

## 后果

### 正面

- 实现简单，易于理解
- 支持跳转到任意页
- `hasMore` 字段简化客户端分页逻辑
- `total` 字段支持总数展示

### 负面

- 大数据量时 offset 性能较差
- 不适合实时数据流（数据变化导致页偏移）

### 迁移路径

未来可添加游标分页作为替代方案，保持向后兼容。
