# ADR-002: JSON 文件作为默认持久化层

## 状态

已采纳

## 背景

Release Guardian 需要一个持久化层来存储发布记录、审批状态和治理数据。选项包括：

- 关系型数据库（PostgreSQL、MySQL）
- 文档数据库（MongoDB）
- 文件存储（JSON、SQLite）
- 内存存储

## 决策

默认使用 JSON 文件作为持久化层，通过原子写入（写入临时文件 + 重命名）保证数据一致性。

## 后果

### 正面

- 零外部依赖，无需额外的数据库服务
- 部署简单，数据文件可直接备份
- 开发和测试环境设置简单
- 数据格式人类可读，便于调试

### 负面

- 不适合高并发写入场景
- 全量加载数据到内存，不适合大数据集
- 单实例限制

### 迁移路径

项目提供了详细的数据库迁移文档（`docs/DATABASE-MIGRATION.md`），包含 PostgreSQL schema 设计和适配器模式，便于生产环境升级。
